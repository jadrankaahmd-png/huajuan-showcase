## 核心概念

### 1. Heartbeat 机制

**定义**：定期唤醒 Agent 执行 5 件事

**5 件事**：
1. **回顾** - 回顾最近的工作和决策
2. **日记** - 记录重要事件和学习
3. **扫描** - 扫描环境和数据变化
4. **健康检查** - 检查系统状态和数据完整性
5. **刷新状态板** - 更新状态显示和指标

**对花卷的应用**：
- ✅ 已有 HEARTBEAT.md
- ✅ 定期检查任务（邮箱、日历、天气）
- ✅ 自动健康检查（Redis 数据验证）
- ✅ 刷新统计数字

---

### 2. 渐进式上下文加载

**定义**：三层记忆，按需加载

**三层记忆**：
1. **当前状态** - 实时数据（股票价格、新闻）
2. **短期记忆** - 最近对话和任务
3. **长期知识** - 知识库和历史数据

**核心原则**：
- ❌ 不把所有上下文塞进去
- ✅ 按需加载，节省 Token
- ✅ 分层管理，清晰明了

**对花卷的应用**：
```
当前状态：
  - Redis 实时数据
  - Telegram 新闻流
  - QVeris 行情

短期记忆：
  - 最近对话（MEMORY.md）
  - 今日任务（2026-03-10.md）
  - 最近学习

长期知识：
  - knowledge_base/（31个文件）
  - SQLite 能力库（612条）
  - 历史数据
```

---

### 3. 系统变稳靠删

**定义**：删除低价值自动化，保持清晰度

**核心原则**：
- ❌ 不要添加太多功能
- ✅ 定期清理无用能力
- ✅ 保持系统简洁
- ✅ 只保留核心价值

**对花卷的应用**：
1. **定期清理**
   - 删除重复能力
   - 移除过时知识
   - 精简自动化脚本

2. **价值评估**
   - 评估能力使用频率
   - 删除低频能力
   - 保留核心能力

---

### 4. Harness 复利

**定义**：模型会变，但基础设施会复利

**复利元素**：
1. **文件结构** - 稳定的目录结构
2. **状态同步** - Redis 同步机制
3. **记忆分层** - 三层记忆架构
4. **Heartbeat** - 定期唤醒机制

**核心价值**：
- ✅ 长期价值
- ✅ 可持续发展
- ✅ 模型无关

**对花卷的应用**：
```
文件结构：
  - SOUL.md（人格）
  - MEMORY.md（长期记忆）
  - HEARTBEAT.md（定期任务）
  - data/（数据）
  - scripts/（脚本）
  - public/knowledge_base/（知识库）

状态同步：
  - Redis 作为唯一数据源
  - Git 自动同步（pre-push hook）
  - npm run sync 一键同步
```

---

### 5. 人格层（SOUL.md）

**定义**：定义 Agent 身份和决策风格

**核心原则**：
1. **明确身份** - 我是谁
2. **决策风格** - 如何做决策
3. **不确定性** - 遇到不确定就说不确定
4. **人格一致** - 保持言行一致

**对花卷的应用**：
```markdown
# SOUL.md 已有内容

- **明确身份**：花卷，金融分析助手
- **决策风格**：基于数据和逻辑
- **不确定性**：没有数据时明确说明
- **人格一致**：始终专业、客观
```

---

**核心原则**：
