### **2. Telegram API 缓存问题修复（03:20 - 03:25）**

#### **问题根源**
- Vercel 默认缓存 API 响应
- 即使 Redis 更新，API 还是返回旧数据
- 页面显示 5 小时前的旧数据

#### **修复内容**
```typescript
// 禁用 Next.js 缓存
export const revalidate = 0;

// 禁用浏览器缓存
headers: {
  'Cache-Control': 'no-store, no-cache, must-revalidate',
  'Pragma': 'no-cache',
  'Expires': '0',
}
```

#### **结果**
- ✅ 每次请求都从 Redis 读取最新数据
- ✅ 用户刷新页面能看到最新新闻
- ✅ Redis 有 91 条最新新闻
