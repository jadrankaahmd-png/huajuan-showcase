## 1. 双 Prompt Stack 系统

### 核心概念

**双 Prompt Stack**：Claude 和 GPT 各一套 prompt，每晚自动同步两套 stack，防止 prompt drift（提示词漂移）。

### 关键技术

**LLM 路由器**：
- 按任务类型分配模型
- 简单任务用便宜模型（节省成本）
- 复杂任务用强大模型（保证质量）

### 对花卷的应用

**任务分类**：
```yaml
简单查询：
  - 股票价格查询
  - 基本数据检索
  - 简单计算
  模型：GLM-4（便宜）

深度分析：
  - 财报分析
  - 行业研究
  - 投资决策
  模型：Claude（强大）
```

**自动同步**：
- 每晚自动同步两套 prompt stack
- 确保 Claude 和 GPT 的 prompt 一致
- 防止 prompt drift

## 2. 多层安全架构

### 核心概念

**多层安全架构**：每晚"安全委员会"自动审计，三层防御架构。

### 三层防御

**第一层：数据审计**
- Redis 数据完整性验证
- 数据备份和恢复
- 异常数据检测

**第二层：操作审计**
- Git 操作审计
- 敏感操作审批
- 操作日志记录

**第三层：信息保护**
- 敏感信息加密
- API Key 保护
- 数据泄露防护

### 对花卷的应用

**每晚自动审计**：
```bash
# 1. Redis 数据审计
node scripts/audit-redis.js

# 2. Git 操作审计
git log --since="1 day ago" --oneline

# 3. 敏感信息检查
grep -r "API_KEY\|TOKEN\|SECRET" .
```

**安全委员会**：
- 自动生成审计报告
- 发现异常立即通知
- 提供修复建议

## 3. 知识库 + 内容管线

### 核心概念

**知识库 + 内容管线**：在群里丢一个链接，AI 自动抓取全文、打标签、做向量索引。下次聊到相关话题能调出来。

### 关键技术

**自动抓取**：
- 检测链接
- 抓取全文内容
- 提取关键信息

**智能标签**：
- 自动分类
- 关键词提取
- 主题识别

**向量索引**：
- 内容向量化
- 语义搜索
- 相似度计算

**去重机制**：
- 检测相似内容
- >40% 相似就拦住
- 防止重复存储

**选题卡生成**：
- 搜知识库
- 查社交媒体热度
- 历史内容去重
- 输出完整选题卡

### 对花卷的应用

**自动学习流程**：
```
用户分享链接
  ↓
AI 自动抓取全文
  ↓
打标签 + 向量索引
  ↓
检测相似度（>40%拦住）
  ↓
存入知识库
  ↓
下次对话自动调出
```

**选题卡生成流程**：
```
用户说关键词
  ↓
搜知识库相关内容
  ↓
查社交媒体热度
  ↓
历史内容去重
  ↓
输出完整选题卡
```
