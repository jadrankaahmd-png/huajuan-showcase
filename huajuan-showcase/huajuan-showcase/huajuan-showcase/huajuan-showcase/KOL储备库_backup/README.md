## 🛠️ Twitter监控系统（三重保障）

### 监控方案

| 方案 | 工具 | 优先级 | 说明 |
|------|------|--------|------|
| **方案1** | Nitter + RSS | 主方案 | 通过Nitter RSS获取推文 |
| **方案2** | RSS订阅 | 备用方案 | 纯RSS方式，最稳定 |
| **方案3** | ntscraper | Python方案 | Python库直接抓取 |

### 监控脚本

**主监控器：** `tools/twitter_kol_monitor.py`
- 三重保障 fallback 机制
- 自动切换不同方案
- 支持关键词搜索

**备用监控器：** `tools/rss_kol_monitor.py`
- 纯RSS方案
- 情绪分析功能
- 报告生成

**RSS配置：** `KOL储备库/kol_rss_feeds.yaml`
- 44位KOL的RSS订阅链接
- 关键词配置
- 监控参数设置

### 使用方法

```bash
# 1. 激活环境
source /opt/homebrew/Caskroom/miniconda/base/etc/profile.d/conda.sh
conda activate openclaw

# 2. 运行主监控器（三重保障）
cd ~/.openclaw/workspace/tools
python3 twitter_kol_monitor.py --all

# 3. 运行备用RSS监控器
python3 rss_kol_monitor.py --all

# 4. 搜索关键词
python3 twitter_kol_monitor.py --search NVDA TSLA AI

# 5. 监控单个KOL
python3 twitter_kol_monitor.py nvidia
```

### 监控频率
- **日常监控：** 每小时一次
- **重点KOL：** 每30分钟一次
- **关键词预警：** 实时

### 数据存储
- **位置：** `~/.openclaw/workspace/KOL监控数据/`
- **格式：** JSON
- **保留：** 30天
