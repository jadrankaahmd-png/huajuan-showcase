---

## 核心定位

OpenAI Symphony 是一个自主 Coding Agents 调度服务，监听项目管理工具（Linear 等），为每一个 Issue 自动分配隔离的工作区，启动 Codex 去实现该 Issue，直到任务完成或流转至人工审核。

---

## 六层架构设计

### 1. Policy Layer（策略层）
- 定义Agent行为规则
- 任务优先级策略
- 资源分配策略

### 2. Configuration Layer（配置层）
- WORKFLOW.md 配置文件
- 随代码版本控制
- 实时监听变更无需重启

### 3. Coordination Layer（协调层）
- 任务分发与调度
- 多Agent协调
- 依赖关系管理

### 4. Execution Layer（执行层）
- Agent实际运行
- 隔离工作区
- 任务执行监控

### 5. Integration Layer（集成层）
- 项目管理工具集成（Linear等）
- 代码仓库集成
- 第三方服务集成

### 6. Observability Layer（可观测性层）
- 任务状态追踪
- Agent性能监控
- 运行日志分析

---

## 核心机制

### 并发控制
- **全局上限**：10个Agent并发
- **细粒度限制**：按Issue状态控制
- **停滞检测**：5分钟自动重试
- **单次续跑**：最多20轮

### WORKFLOW.md
```markdown
# 放进代码仓库本身随代码版本控制
# Symphony实时监听变更无需重启
```

**优势**：
- ✅ 配置与代码同步
- ✅ 版本控制
- ✅ 热更新
- ✅ 无需重启服务

### 停滞检测与重试
- **检测周期**：5分钟
- **自动重试**：任务超时自动恢复
- **失败处理**：流转至人工审核
