## 🎯 核心要点

### **1. Skill 文件夹结构（标准）**

```
skill-name/
├── SKILL.md          # 指令（必须）
├── scripts/          # 代码（可选）
│   ├── main.js       # 主脚本
│   └── utils.js      # 工具函数
├── references/       # 文档（可选）
│   ├── api-docs.md   # API 文档
│   └── examples.md   # 示例
└── assets/           # 模板（可选）
    ├── template.md   # 模板文件
    └── config.json   # 配置文件
```

**花卷现状**：
- ✅ SKILL.md（必须）- 已实现
- ✅ scripts/（代码）- 已实现
- ⚠️ references/（文档）- 部分实现
- ❌ assets/（模板）- 未实现

### **2. 描述字段是关键（最关键）**

**官方标准**：
```yaml
---
name: skill-name
description: 具体描述技能功能。当用户说"触发短语1"、"触发短语2"时使用。触发条件：具体条件。
---
```

**最佳实践**：
- ✅ **描述功能**：清楚说明技能做什么
- ✅ **用户触发短语**：至少2-3个用户可能说的短语
- ✅ **触发条件**：明确什么时候应该触发

**示例（官方推荐）**：
```yaml
---
name: sprint-planner
description: 帮助团队规划冲刺。当用户说"规划我的冲刺"、"创建冲刺计划"、"安排这周的工作"时使用。触发条件：用户提到"冲刺"、"sprint"、"规划工作"。
---
```

### **3. 技能三层加载（架构设计）**

**官方标准**：
- **第1层：菜单封面（始终可见）** - 简短描述，触发短语
- **第2层：完整菜单（按需加载）** - 详细说明，执行步骤
- **第3层：厨房（特定触发）** - 完整代码，实现细节

**花卷应用**：
- ✅ **第1层**：description 字段（简短描述）
- ✅ **第2层**：SKILL.md 内容（详细说明）
- ✅ **第3层**：scripts/ 文件夹（实现代码）

---

### **4. 三种技能类型**

#### **创建技能（Creation Skills）**
- **作用**：帮助用户创建新内容
- **示例**：创建文档、生成代码、写报告
- **触发短语**："创建"、"生成"、"写一个"

#### **工作流程技能（Workflow Skills）**
- **作用**：自动化多步骤流程
- **示例**：代码审查、测试流程、部署流程
- **触发短语**："运行"、"执行"、"检查"

#### **MCP增强技能（MCP-Enhanced Skills）**
- **作用**：集成外部工具和服务
- **示例**：API调用、数据库查询、文件操作
- **触发短语**："调用"、"查询"、"连接"

**花卷 skills 分类**：
- ✅ **code-change-verification** - 工作流程技能
- ✅ **prune-system-prompts** - 工作流程技能
- ✅ **docs-sync** - 工作流程技能
- ✅ **heartbeat-orchestrator** - 工作流程技能

---

### **5. 测试方法（验证可靠性）**

**官方推荐测试**：
1. **应该触发时触发** - 测试触发短语是否有效
2. **不应该触发时不触发** - 测试不会误触发
3. **输出符合预期** - 测试输出格式和内容

**测试用例模板**：
```markdown
## 测试用例

### ✅ 应该触发
- 用户说："验证代码变更"
- 用户说："检查 Redis 数据"
- 用户说："同步能力"

### ❌ 不应该触发
- 用户说："验证身份证"（不相关）
- 用户说："检查邮件"（不相关）
- 用户说："同步手机"（不相关）

### ✅ 预期输出
- 运行 npm run sync
- 验证 Redis 数据完整性
- 验证统计数字准确性
- 生成验证报告
```
